分析有可能是使用了引擎扩展
在a=version显示的phpinfo()的内容中找到配置文件为/usr/local/lib/php.ini

成功获得php.ini

右击查看源代码，ctrl+f搜索extension

注意没有使用;注释的extension=zend_test

观察到
; Note : The syntax used in previous PHP versions ('extension=<ext>.so' and ; 'extension='php_<ext>.dll') is supported for legacy reasons and may be ; deprecated in a future PHP major version. So, when it is possible, please ; move to the new ('extension=<ext>) syntax.
推测文件名是zend_test.so或php_zend_test.dll

读取 /proc/self/maps 可查看当前进程的内存映射关系
a=version 使用ctrl+f查找extension的存放位置

a=read&file=/usr/local/lib/php/extensions/no-debug-non-zts-20190902/zend_test.so读取zend_test.so文件

然后需要进行逆向
使用python脚本下载文件

得到文件zend_test.so
使用IDA打开(格式ELF)
打开后页面右侧有

推测经过了rc4加密

密钥为：abcsdfadfjiweur

可使用index.php文件验证

然后上传RC4加密的webshell
使用 python 写一句话木马保存为shell.php文件，rc4加密后上传

由index界面可知，文件保存路径为/upload/shell.php

  • 使用蚁剑连接
    • 在根目录下发现flag文件（复现环境权限被修改，可以直接打开，参考其他wp发现需要提取）
    • 打开虚拟终端
    • cd /
    • cat flag权限不足
    • 使用ls -al查看所有文件和权限
    • 使用sudo -l提示tty问题
    • 使用python开启pty环境（实践失败）

反弹shell

  • 搭建阿里云服务器
  • 安装 NC
    yum install nmap-ncat
  • 在服务器安全组增加开启 2333 端口 （与监听端口保持一致即可）
  • 开启监听
    nc -lvp 2333
  • post请求提交cmd=”phpinfo()”发现可以成功返回
    提交请求可以使用burpsuite或hackbar
  • 利用 hackbar 插件反弹 shell

  • chmod 提权方法
