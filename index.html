浏览器解码过程

参考：
HTML与JavaScript自解码机制
一次对浏览器解析和XSS的深度探究

为什么要进行编码

  • 主要是因为某些数据不适合传输

    • Size 过大
    • 包含隐私数据
    • 有些字符会引起歧义

  • 对于 URL： &用于分割多个参数。

    • 倘若有某个参数键值为 key=va&lue，会因为key的值va&lue中携带&而造成歧义。
    • url编码后，服务端会把紧跟在“%”后的字节当成普通的字节，就是不会把它当成各个参数或键值对的分隔符。

  • 对于 HTML： 当浏览器遇到<会识别为元素定义的开始，>会识别为元素的结束

    • 倘若有<div id="1>" ></div>，由于标签的属性值携带了>，同样会造成歧义
    • 因此属性值的>需要进行HTML编码(使用字符实体)。

三种编码

HTML编码（字符实体）

  • 字符实体是一个预先定义好的转义序列
  • 两种表示方法：
    1. 字符实体以&+预先定义的实体名称+;，如“<”的实体名称为&lt;
    2. 字符实体还可以以&#+字符在 ASCII 对应的十进制数字+;，如<的实体编号为&#60;

      JavaScript编码

  • Unicode 形式：\uH (十六进制)。
  • 普通十六进制：\xH。
  • 纯转义：’ 、 “ 、 < 、>这样在特殊字符之前加 \ 进行转义。

    URL编码

    RFC3986 文档规定，URL 中只允许包含英文字母（a-zA-Z）、数字（0-9）、-_.~4 个特殊字符以及所有保留字符
    RFC3986 中指定了以下字符为保留字符：! * ‘ ( ) ; : @ & = + $ , / ? # [ ]
  • 编码方式 %加字符在 ASCII 码表中的十六进制值。例如，/在 ASCII 码表中十六进制为 0x2f，那么它对应的 URL 编码为%2f。
  • JavaScript 中提供了 3 个函数用来对 URL 编码以得到合法的 URL：
    1. escape()
    2. encodeURI()
    3. encodeURIComponent()

浏览器的解码规则

  1. HTML 解析器对 HTML 文档进行解析，完成 HTML 解码并且创建 DOM 树
  2. JavaScript 或者 CSS 解析器对内联脚本进行解析，完成 JS、CSS 解码
  3. URL 解码会根据 URL 所在的顺序不同而在 JS 解码前或者解码后
  • 不在<script>标签内（如onclick属性）：
    上下文环境是 HTML，在 JavaScript 执行之前，HTML 形式的编码会自动解码。
    • JavaScript 可以进行 HTML 形式的编码，这种编码有以下两种。
      1. 进制编码：&#xH;（十六进制格式）、&#D;（十进制格式），最后的分号（;）可以不要。
      2. HTML 实体编码：HtmlEncode
  • 在<script>标签内：
    上下文环境是 JavaScript，不是HTML（可以认为标签里的内容和 HTML 环境毫无关系）
    • 此时内容要遵守的是JavaScript 编码，具体有如下几种形式：
      1. Unicode 形式：\uH (十六进制)。
      2. 普通十六进制：\xH。
      3. 纯转义：\’ 、 \” 、 \< 、>这样在特殊字符之前加 \ 进行转义。

Q1：什么是重入码编写的文件？
A: 【参考百度百科：可重入代码】 可重入代码(Reentry code)：也叫纯代码(Pure code)是一种允许多个进程同时访问的代码。 为了使各进程所执行的代码完全相同，故不允许任何进程对其进行修改。程序在运行过程中可以被打断，并由开始处再次执行，并且在合理的范围内（多次重入，而不造成堆栈溢出等其他问题），程序可以在被打断处继续执行，且执行结果不受影响。 若一个程序或子程序可以安全的被并行执行，则称其为可重入（reentrant或re-entrant）的。

Q2：虚拟存储器、虚拟设备技术
A: 【参考百度百科：虚拟存储器】 虚拟存储器(Virtual Memory)：在具有层次结构存储器的计算机系统中，自动实现部分装入和部分替换功能，能从逻辑上为用户提供一个比物理贮存容量大得多，可寻址的“主存储器”。 【参考百度百科：虚拟设备】 虚拟设备：通过虚拟技术将一台独占设备虚拟成多台逻辑设备，供多个进程同时使用， 通常把这种经过虚拟的设备称为虚拟设备。

Q3：操作系统的异步性指什么？
A:【参考百度题库】 异步性也称为不确定性，是指在多个程序并发运行环境中，每个程序何时开始执行、何时暂停、推进速度和完成时间等都是不确定的。

Q4：为什么说操作系统扩充机器？
A:有了操作系统，机器可以发挥更大的作用。

Q5：处理机&处理器
`A:
处理机：

  • 处理机包括中央处理器，主存储器，输入-输出接口，加接外围设备就构成完整的计算机系统。
  • 处理机是处理计算机系统中存储程序和数据，并按照程序规定的步骤执行指令的部件。
    处理器：中央处理器（Central Processing Unit，简称CPU）作为计算机系统的运算和控制核心，是信息处理、程序运行的最终执行单元。`

Q6：广义指令（系统调用？）
A: 系统调用（又称广义指令），用户请求操作系统提供服务，如外部设备的使用、分配和回收内存。

Q1：摩尔定律
百度百科-摩尔定律
A:英特尔创始人之一戈登·摩尔的经验之谈，其核心内容为：集成电路上可以容纳的晶体管数目在大约每经过18个月到24个月便会增加一倍。换言之，处理器的性能大约每两年翻一倍，同时价格下降为之前的一半。
摩尔定律是内行人摩尔的经验之谈，并非自然科学定律，它一定程度揭示了信息技术进步的速度。
归纳起来，主要有以下3种“版本”： 1、集成电路芯片上所集成的电路的数目，每隔18个月就翻一番； 2、微处理器的性能每隔18个月提高一倍，而价格下降一半； 3、用一美元所能买到的计算机性能，每隔18个月翻两番
【价格是另外的因素？】`

Q2：EDVAC机
百度百科-EDVAC
A:离散变量自动电子计算机（英语：Electronic Discrete Variable Automatic Computer，EDVAC）是一台美国早期电子计算机。
与它的前任ENIAC不同，EDVAC采用二进制，而且是一台冯·诺伊曼结构的计算机 可以说EDVAC是第一台现代意义的通用计算机。 整台计算机共使用大约6000个电子管和大约12000个二极管，功率为56KW，占地面积45.5平方米，重7850KG，使用时需要三十个技术人员同时操作。
它由五个基本部分组成：1）运算器CA（2）控制器CC（3）存储器M（4）输入装置I（5）输出装置O；

Q7：相联存储器（关联存储器）
A:一种不根据地址而是根据存储内容来进行存取的存储器，可以实现快速地查找快表。既可以按照地址寻址也可以按照内容寻址（通常是某些字段），为了与传统寄存器作区别，称为按内容寻址的存储器。

Q8：“向后兼容”
A:Backwards compatibility，又称向下兼容（Downward Compatibility），与向上兼容（向前兼容）相对，在计算机中指在一个程序或者类库更新到较新的版本后，用旧的版本程序创建的文档或系统仍能被正常操作或使用，或在旧版本的类库的基础上开发的程序仍能正常编译运行的情况。【让新机器可以运行旧程序】
向上兼容（Upward Compatible）又称向前兼容（Forward Compatible），在某一平台的较低版本环境中编写的程序可以在较高版本的环境中运行。【让旧程序可以在新机器上运行】
